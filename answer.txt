Реактивное программирование отличается от событийного тем, что реактивное подразумевает собой написание кода
для всех частей предложения одновременно. Событийное программирование - это программирование, которое включает в себя написание 
кода для отдельной части программы, создание ее функционала и обеспечение работоспособности функций

API - это особая структура, позволяющая создавать взаимодействие между одной программой и другой. Пользоваться функционалом, копировать
данные, работать в одной среде. Например, каждый бот в Телеграме имеет свой API ключ, позволяющий разработчику создавать новый функционал,
управлять ботом, изменять его поведение на те или иные действия и т.д
